# Ralph Progress Log

## Codebase Patterns
- CLI args: clap derive API in `src/cli.rs`, validation in `validate_args()` exits with code 2
- `--flash` is a string value arg ("True"/"False"), NOT a bare flag — use `value_parser`
- `--diarization_model` needs `#[arg(long = "diarization_model")]` to preserve underscore
- `--language` default is literal string `"None"`, checked via `is_auto_detect_language()`
- `.en` model detection: `model_name.split('.').next_back() == Some("en")`
- Dead code: use `#[allow(dead_code)]` on module declaration in main.rs for functions used in later stories
- clippy: use `next_back()` instead of `last()` on `Split` iterators
- Audio decoding: symphonia with `MediaSourceStream`, probe format, decode packets into `SampleBuffer<f32>`
- Audio resampling: linear interpolation in `resample()`, mono conversion in `to_mono()`
- Audio chunking: `chunk_audio()` splits into 30s (480000 sample) chunks, zero-pads last chunk
- Convert subcommand: `run_convert()` in `src/convert.rs` reads JSON, converts to SRT/VTT/TXT, output filename is always `output.<format>`
- Audio from bytes: `load_audio_from_bytes()` accepts `Vec<u8>` for URL-downloaded audio
- URL detection: `is_url()` checks `starts_with("http://")` or `starts_with("https://")`
- Audio download: `download_audio()` uses reqwest blocking client with indicatif progress bar
- Integration tests: `assert_cmd` + `predicates` crates for CLI binary testing
- WAV generation: RIFF header + fmt (PCM, channels, sample_rate, 16-bit) + data (i16 samples)
- Flash Attention: compile-time via `flash-attn` cargo feature; runtime `--flash True` warns if not compiled
- Unified loading: `load_audio()` dispatches to file or URL path automatically
- Model loading: `hf-hub` ApiBuilder with `.with_token()` for auth, `Api::new()` for unauthenticated
- Device selection: `select_device()` handles "mps", numeric CUDA IDs, CPU fallback (cfg-gated)
- Mel filters: embedded via `include_bytes!` from `assets/melfilters{,128}.bytes`
- Model weights: `VarBuilder::from_mmaped_safetensors` with F16 on GPU, F32 on CPU
- WhisperModel struct: holds model, config, tokenizer, device, mel_filters together
- Mel spectrogram: `m::audio::pcm_to_mel(config, samples, filters)` returns flat `Vec<f32>`, reshape to (1, n_mel, n_frames)
- Mel tensor dtype: F16 on GPU, F32 on CPU (must match model weights dtype)
- Decoder flow: `decoder.forward(tokens, audio_features, flush_kv_cache)` -> `decoder.final_linear(ys)` -> logits
- KV cache: first call uses flush=true with full prompt, subsequent calls use flush=false with single token
- Timestamp tokens: `no_timestamps_token + 1` is timestamp_begin; each increment = 0.02 seconds
- Token suppression: `config.suppress_tokens` contains token IDs to mask during decoding
- Language tokens: `<|en|>`, `<|fr|>`, etc. — looked up via tokenizer
- Special tokens: SOT_TOKEN, EOT_TOKEN, TRANSCRIBE_TOKEN, TRANSLATE_TOKEN, NO_TIMESTAMPS_TOKEN in `m::` module
- Batched encoding: stack mel tensors with `Tensor::cat(&mels, 0)`, run encoder once, extract per-item with `.i(i..i+1)`
- Decoder is per-sequence (KV cache): must reset and decode each batch item sequentially after batched encoding
- `transcribe()` in inference.rs: full pipeline — chunk audio, batch encode, sequential decode, progress bar
- Output JSON: `build_output()` creates serde Value, `write_output()` writes to file, `print_success_message()` for stdout
- serde_json `preserve_order` feature needed for JSON key ordering (speakers, chunks, text)
- Timestamp type: tuple struct `Timestamp(f64, Option<f64>)` serializes as `[f64, f64]` or `[f64, null]`
